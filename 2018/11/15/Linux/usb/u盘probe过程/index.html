u盘probe过程

发表于 2018-11-15 | 分类于 Linux ， usb |

匹配方式

一般u盘都是使用usb_storage_usb_ids中的USUAL_DEV(USB_SC_SCSI, USB_PR_BULK)匹配成功的。

lsusb -v

USUAL_DEV(USB_SC_SCSI, USB_PR_BULK)

以上，所以匹配。

代码分析

插入一个usb设备，有两次probe过程。
第一次是usb device的probe，第二次是usb interface的probe，usb_device_match会执行两次。

注册了两种driver

都是挂载在usb_bus_type上

usb_generic_driver

struct usb_device_driver usb_generic_driver;

usb_storage_driver

static struct usb_driver usb_storage_driver;

第一次probe过程

  1. 创建dev

  2. match
    调用usb_bus_type.match函数，因为usb_device_type，所以进if分支

  3. probe
    match成功，调用usb_generic_driver.probe。
    generic_probe中又创建了dev，所以引发第二次probe

第二次probe过程

  1. match
    调用usb_bus_type.match函数，因为usb_if_device_type，所以进else分支

  2. probe
    调用usb_storage_driver.probe
